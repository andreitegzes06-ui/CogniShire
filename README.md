# Human-Computer Interaction

Human-Computer Interaction studies how people understand, use, design, evaluate, and live with interactive computer systems.

This collection focuses on five HCI areas:

1. Understanding the User
2. System and Interaction Design
3. Evaluating the Design
4. Accessibility and Inclusive Design
5. Human-AI Interaction

The material connects HCI to cognitive science, software engineering, psychology, accessibility, ethics, artificial intelligence, research methods, and academic venues.

## Core Questions

- How do users perceive, think, remember, decide, and form mental models?
- How are interfaces, interaction flows, prototypes, feedback, and visual structures designed?
- How can designs be evaluated with evidence?
- How can systems avoid excluding people with different abilities, tools, contexts, and access needs?
- How should people understand, verify, trust, control, and remain responsible when using AI systems?

## Academic Anchors

The notes use CS2023 and public HCI resources as academic anchors. Relevant communities and sources include ACM CHI, ASSETS, CSCW, IUI, FAccT, W3C WAI, WCAG, Nielsen Norman Group, Microsoft Human-AI guidelines, Google PAIR, and related peer-reviewed HCI work.
