# Cognishire: A Map of Human-Computer Interaction

Cognishire is an Obsidian learning vault about **Human-Computer Interaction**. It is built as a guided study map for students who want to understand HCI as an academic and professional field.

The project uses a light fantasy map style to make navigation memorable, but the main purpose is academic. Each area explains real HCI concepts, connects them to CS2023 topics, and gives students a practical route for learning, research, and portfolio work.

## Purpose

This vault helps students explore HCI through five connected areas:

1. **Mind Library**  
    Understanding the User
    
2. **Interface Forge**  
    System and Interaction Design
    
3. **Observation Chamber**  
    Evaluating the Design
    
4. **Inclusive Gate**  
    Accessibility and Inclusive Design
    
5. **Oracle Engine**  
    Human-AI Interaction
    

Each area includes pages such as:

- Overview
    
- Theory
    
- Design
    
- Experiment
    
- Connections
    
- Important People
    
- Important Venues
    
- Local and Global
    
- Open Problems
    

## What the project teaches

Cognishire introduces students to core HCI questions:

- How do users think, perceive, remember, and act?
    
- How are interfaces planned and designed?
    
- How can designs be evaluated with evidence?
    
- How can systems become more accessible and inclusive?
    
- How should humans interact with AI systems responsibly?
    

The vault also connects HCI to related areas such as cognitive science, software engineering, psychology, accessibility, ethics, AI, research methods, and academic career planning.

## Local and global perspective

The project is written with a local and global structure.

The local layer connects HCI to the **UVT Faculty of Informatics** context, student projects, Obsidian, GitHub, classroom use, and academic review.

The global layer connects the material to recognised HCI sources and communities, including CS2023, ACM CHI, ASSETS, CSCW, IUI, FAccT, W3C WAI, WCAG, Nielsen Norman Group, Microsoft Human-AI guidelines, Google PAIR, and other academic or professional references.

## How to use the vault

1. Download or clone this repository.
    
2. Open the folder as an Obsidian vault.
    
3. Start from:
    

```text
Human-Computer Interaction.md
```

4. Follow the five room links from the main entrance page.
    
5. Read each room in this order:
    

```text
Overview
Theory
Design
Experiment
Connections
Important People
Important Venues
Local and Global
Open Problems
```

## Recommended setup

For the intended visual experience, open the project in Obsidian.

The vault includes Obsidian configuration files, theme settings, CSS snippets, images, and Markdown pages. Some diagrams and visual styling may look different in GitHub preview compared with Obsidian.

If the styling does not appear correctly in Obsidian:

1. Open Obsidian settings.
    
2. Go to **Appearance**.
    
3. Enable the included CSS snippets.
    
4. Use the included theme settings if available.
    
5. Reopen the vault in Reading View.
    

## Repository structure


## Project status

The vault is still being refined. Some pages may continue to be improved for:

- clearer academic writing
    
- stronger source grounding
    
- better internal links
    
- better accessibility
    
- cleaner diagrams

## Academic note

This project is a learning map, not an official curriculum. It uses CS2023 and public HCI resources as academic anchors, but it should be read as a student-built guide.

Claims about people, venues, methods, and institutions should be checked against official sources when used in formal academic work.
